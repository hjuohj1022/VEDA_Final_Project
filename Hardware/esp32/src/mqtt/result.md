# 열화상 데이터 전송 최적화 및 시스템 안정화 보고서 (최종)

## 1. 해결된 핵심 문제
- **가로 찢어짐 현상 해결:** 세그먼트 동기화 로직을 개선하여 0~19번 패킷의 누락을 방지하고 전체 프레임(160x120)을 온전히 복원.
- **화면 밀림 및 색상 왜곡 해결:** Teensy에서 전송 전 바이트 순서(Endian)를 교정하여 수신 측(Python)에서의 데이터 해석 오류 차단.
- **UART 통신 안정화:** 고속(921,600bps) 통신 시 발생하는 버퍼 오버플로우 및 데이터 누락을 분할 전송(Flow Control)으로 해결.

## 2. 주요 개선 및 변경 사항

### [Teensy 4.1] - 데이터 수집 및 전송 최적화
- **Predictive Segment Indexing:** 20번 패킷에서 세그먼트 번호를 확인한 후, 다음 세그먼트의 0~59번 패킷 위치를 미리 계산하여 저장하는 예측 로직 적용.
- **Big-Endian 변환:** Teensy(Little-Endian) RAM에 저장할 때 미리 바이트를 뒤집어, Python/ESP32가 별도 변환 없이 Big-Endian으로 처리하도록 수정.
- **UART Chunked Send:** 38,400 바이트를 한 번에 쏟아내지 않고, 1KB 단위로 쪼개어 전송하며 중간에 `100us`의 휴식을 주어 ESP32의 수신 부하 경감.
- **Flush 최적화:** 반복문 내의 `flush()`를 제거하고 전체 전송 후 한 번만 수행하여 전송 속도 극대화.

### [ESP32-C3] - UART 수신 및 디버깅 강화
- **수신 로직 보강:** `uart_get_buffered_data_len`을 사용하여 버퍼에 쌓인 데이터를 한 번에 대량으로 읽어 처리 효율 향상.
- **타임아웃 유연화:** 데이터 수신 중 발생하는 미세한 지연을 허용하도록 타임아웃을 `100ms`로 상향 조정하여 프레임 포기 현상 감소.
- **상태 모니터링:** `>> Header Found!`, `>> Frame Received!` 로그를 통해 하드웨어 간 동기화 상태를 실시간으로 확인 가능하게 개선.

### [Python Client] - 수신 안정성 및 렌더링 개선
- **버퍼 격리 조립:** 청크가 모두 모인 순간 데이터를 딕셔너리로 복제(Deep Copy)하여 처리함으로써, 렌더링 중 다음 프레임 데이터가 유입되어 발생하는 데이터 오염 방지.
- **데이터 클리핑:** Lepton 3.5의 유효 범위를 넘어서는 노이즈(30,000 이상)를 `np.clip`으로 제거하여 화면 깨짐 방지.
- **실시간 로그:** `📦 [idx/total]` 형식을 통해 데이터 수신 진행률을 가시화.

## 3. 최종 시스템 프로토콜
- **Header:** `FSTART` (6 bytes)
- **Offset:** `0` (1 byte, 고정)
- **Data:** `160 * 120 * 2` bytes (Big-Endian RAW14)
- **MQTT Topic:** `lepton/frame/chunk` (8-byte header 포함)

## 4. 현재 상태 및 결과
- **프레임 품질:** 가로 찢어짐 없이 160x120 전체 화면이 깨끗하게 출력됨.
- **전송 속도:** 약 1~2 FPS (MQTTS 보안 연결 및 921,600bps UART 환경).
- **안정성:** 장시간 구동 시에도 MQTT 연결 유지 및 데이터 동기화 안정적.
