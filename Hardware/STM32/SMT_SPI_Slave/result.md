# 모터 제어 방식 변경 결과 보고서

## 1. 개요
기존의 클라이언트 의존적 각도 이동 방식에서, 버튼을 누르고 있는 동안 1도씩 연속적으로 움직이고 버튼을 떼면 멈추는 **상태 기반 제어 방식**으로 변경하였습니다.

## 2. 주요 변경 사항

### 2.1. `motor.h` (API 정의 추가)
*   **상태 업데이트 함수 추가**: `Motor_Update(void)`를 정의하여 메인 루프에서 주기적으로 호출될 수 있도록 함.
*   **연속 이동 제어 함수 추가**: `Motor_StartMove()`, `Motor_Stop()` 함수를 추가하여 모터의 이동 상태를 관리.
*   **명령어 형식 정의**: `press`, `release` 키워드를 포함하는 새로운 명령 형식을 주석에 반영.

### 2.2. `motor.c` (제어 로직 구현)
*   **상태 변수 도입**: `s_moving[MOTOR_NUM]` 배열을 통해 각 모터의 이동 방향(-1: 왼쪽, 1: 오른쪽, 0: 정지)을 관리.
*   **시간 기반 업데이트**: `HAL_GetTick()`을 사용하여 20ms마다 1도씩 각도를 변경하도록 구현하여 부드러운 움직임 제공.
*   **명령어 파서 고도화**: 
    *   `motor1 left press`: 왼쪽 이동 시작
    *   `motor1 right press`: 오른쪽 이동 시작
    *   `motor1 release` / `motor1 left release`: 이동 중지
    *   기존의 `set <deg>` 및 상대 이동 명령과의 하위 호환성 유지.

### 2.3. `main.c` (시스템 루프 수정)
*   **비차단식 구조**: `while(1)` 루프 상단에서 `Motor_Update()`를 지속적으로 호출.
*   **통신 효율 개선**: `HAL_SPI_TransmitReceive`의 타임아웃을 5000ms에서 **10ms**로 대폭 단축하여, SPI 통신 대기 중에도 모터 제어 로직이 멈추지 않도록 개선.

## 3. 사용 방법 (명령어 예시)
| 동작 | 명령어 | 비고 |
| :--- | :--- | :--- |
| 왼쪽으로 계속 이동 | `motor1 left press` | 버튼을 누를 때 전송 |
| 오른쪽으로 계속 이동 | `motor1 right press` | 버튼을 누를 때 전송 |
| 이동 중지 | `motor1 release` | 버튼을 뗄 때 전송 |
| 특정 각도로 이동 | `motor1 set 90` | 절대 각도 이동 |

## 4. 기대 효과
*   사용자 입력에 대한 즉각적인 반응성 확보.
*   서보 모터의 부드러운 가감속(1도 단위) 제어 가능.
*   통신 타임아웃 최소화를 통한 시스템 안정성 향상.
